Lockfree

Why Lockfree

The typical motivations given for using lockfree programming are as follows:

  • No lock contention - Theads don't wait to aquire a lock. Any contention will happen only on a single atomic variable which is usually much cheaper than acquiring a lock as it avoids kernel involvement (sleeping, waking, context switching etc.).
  • Progress Guarantees - at least one thread will always be making progress. With locks, we have issues such as deadlocking, priority inversion and convoying (slow threads holding up other threads)
  • Scaling concurrency - without large critical sections where only one thread can do useful work, lock free programming can scale better with more concurrency.

These advantages are all well and good, however there are some domains that live and die by millisecond latency. If latency spikes kill your application, lock free data structures are your friend.

3 Lock Free Data Structures

Treiber Stack

The treiber stack uses compare and swap to push and pop nodes

push

Push is essentially a compare and swap on the old head and new head. 

std::atomic<Node*> head{nullptr};

void push(int value) {
    Node* new_node = new Node{value, nullptr};

    Node* old_head;

    old_head = head.load();
    new_node->next = old_head;
    head.compare_exchange_strong(old_head, new_node);
}

pop

Pop is a compare and swap on the current head and head->next. If head hasn't changed during the CAS, set head->next as head and return the old head. 

Node* pop() {
    Node* old_head;
    Node* new_head;

    old_head = head.load(std::memory_order_acquire);
    if (old_head == nullptr)
        return nullptr;
    new_head = old_head->next;
    head.compare_exchange_strong(old_head, new_head, std::memory_order_acquire, std::memory_order_relaxed);
    return old_head;
}

The ABA problem

Compare and swap is just checking pointers. This can lead to the following problem: * Thread 1 is in the middle of a pop for node A (the current head) and gets paused by the scheduler just after new_head = old_head->next; * Thread 2 pops node A and node B, then pushes a new node to the same memory address as A * Thread 1 thinks nothing has changed in the CAS because the head is still at A, so pops A and makes B the head. B was already freed by thread 2 and so the stack is in a corrupted state. B can point to freed memory. Three common solutions to this are as follows:

tagged pointers

Instead of performing CAS operations on raw pointers, we use a tagged pointer and increment the tag value every time the head node is updated. This ensures the pointer is unique, even if it addresses the same memory. 

    struct TaggedPtr {
        Node* ptr;
        uint64_t tag;
    };
Our tagged pointer is now 128 bits wide so we need double width CAS, or we can fit our tagged pointer into one word. Tagged pointers don't prevent use-after-free, so in practice hazard pointers are preferred.

hazard pointers

Keep a record of 'in use' pointers. Don't free a pointer if it is still in use. This solves the use-after-free issue that we get with ABA. On pop, we retire the node by adding it to a 'retire' list instead of freeing it immediately. We then periodically clear the 'retire' list by iterating through and checking that the node to be retired is not currently in use.

epoch-based reclaimation

Maintain a global epoch counter. Once all threads are in the current epoch, advance the epoch. When a thread is finished with a node, add it to the retire list along with the current epoch value. If a node is >=2 epochs old, it is safe to delete since no other threads can be using it. Check if we can delete nodes in the retired list each time we add to it and it has reached some threshold size.

Michael Scott Queue (FIFO MPMC Lockfree Queue)

fifo queue supports multi producer, multi consumer

maintain head and tail pointer and use dummy node at the front of the queue to simplify enqueue/dequeue logic

The tail pointer may not always be pointing to the very last node which is a design choice that significantly improves throughput of the algorithm.

#include <atomic>

template <typename T>
class MSQueue {
private:
    struct Node {
        T value;
        std::atomic<Node*> next;

        Node(T val) : value(val), next(nullptr) {}
    };

    std::atomic<Node*> head;
    std::atomic<Node*> tail;

public:
    MSQueue() {
        Node* dummy = new Node(T{}); // dummy node
        head.store(dummy);
        tail.store(dummy);
    }

    ~MSQueue() {
        // Not safe in concurrent use — just for completeness
        while (dequeue().has_value()) {}
        delete head.load();
    }

    // Enqueue
    void enqueue(T value) {
        Node* new_node = new Node(value);

        while (true) {
            Node* last = tail.load(std::memory_order_acquire);
            Node* next = last->next.load(std::memory_order_acquire);

            if (last == tail.load(std::memory_order_acquire)) {

                // Case 1: tail is really the last node
                if (next == nullptr) {
                    // Try to link new node
                    if (last->next.compare_exchange_weak(
                            next,
                            new_node,
                            std::memory_order_release,
                            std::memory_order_relaxed)) {

                        // Try to swing tail to new node (optional)
                        tail.compare_exchange_weak(
                            last,
                            new_node,
                            std::memory_order_release,
                            std::memory_order_relaxed);

                        return;
                    }
                }
                // Case 2: tail is lagging → help advance it
                else {
                    tail.compare_exchange_weak(
                        last,
                        next,
                        std::memory_order_release,
                        std::memory_order_relaxed);
                }
            }
        }
    }

    // Dequeue
    std::optional<T> dequeue() {
        while (true) {
            Node* first = head.load(std::memory_order_acquire);
            Node* last  = tail.load(std::memory_order_acquire);
            Node* next  = first->next.load(std::memory_order_acquire);

            if (first == head.load(std::memory_order_acquire)) {

                // Case 1: queue looks empty
                if (first == last) {
                    if (next == nullptr) {
                        return std::nullopt; // empty
                    }

                    // Tail is behind → help it
                    tail.compare_exchange_weak(
                        last,
                        next,
                        std::memory_order_release,
                        std::memory_order_relaxed);
                }
                else {
                    // Read value BEFORE CAS
                    T value = next->value;

                    // Try to move head forward
                    if (head.compare_exchange_weak(
                            first,
                            next,
                            std::memory_order_release,
                            std::memory_order_relaxed)) {

                        // MEMORY LEAK HERE (no reclamation)
                        // delete first;  <-- unsafe without hazard pointers / EBR

                        return value;
                    }
                }
            }
        }
    }
};

enqueue

Enqueue has two cases to handle. If tail->next == null then tail is the last node. We do two CAS operations, one to add the node at the end, and one to set the tail pointer to the new node.

If tail->next != null then tail doesn't actually point to the last node. Try advancing the tail and then retry the enqueue operation.

SPSC Ring Buffer

For the single producer single consumer, we use release acquire semantics with load and store

Apple silicon uses 128 byte cache lines. Using alignas(128) ensures the two variables, full and slot do not sit on the same cache line.

If they were to land on the same cache line, the CPU would have to synchronize access from different cores, adding CPU cycles. This is known as false sharing.


struct RingBufferLockFree {
    alignas(128) std::atomic<bool> full{false};
    char pad1[128 - sizeof(std::atomic<bool>)];
    alignas(128) std::atomic<uint64_t> slot{};
    char pad2[128 - sizeof(std::atomic<uint64_t>)];

    void push(uint64_t v) {
        while (full.load(std::memory_order_acquire))
            CPU_PAUSE();
        slot.store(v, std::memory_order_relaxed);
        full.store(true, std::memory_order_release);
    }

    bool pop(uint64_t& out) {
        if (!full.load(std::memory_order_acquire))
            return false;
        out = slot.load(std::memory_order_relaxed);
        full.store(false, std::memory_order_release);
        return true;
    }
};
